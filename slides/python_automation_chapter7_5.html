행동 3: 요소 위치찾기

웹페이지 자동화의 핵심은 원하는 요소를 정확하게 찾는 것입니다. Selenium은 다양한 방법으로 웹페이지의 요소를 찾을 수 있는 기능을 제공합니다.

요소 찾기 메서드

Selenium에서는 크게 두 가지 형태의 요소 찾기 메서드를 제공합니다:

메서드 설명
find_element() 조건에 맞는 첫 번째 요소만 반환
find_elements() 조건에 맞는 모든 요소를 리스트로 반환
# ChatGPT: "셀레니움으로 요소 찾기 기본 코드"
from selenium import webdriver
from selenium.webdriver.common.by import By

# 첫 번째 매칭 요소 찾기
element = driver.find_element(By.ID, "login-button")

# 모든 매칭 요소 찾기
elements = driver.find_elements(By.CLASS_NAME, "product-item")

# 찾은 요소의 수 확인
print(f"총 {len(elements)}개의 상품을 찾았습니다.")

선택자 활용 예제

# ChatGPT: "여러 선택자를 사용한 요소 찾기 예제"
# ID로 요소 찾기
login_button = driver.find_element(By.ID, "login")

# 클래스명으로 요소 찾기
nav_items = driver.find_elements(By.CLASS_NAME, "nav-item")

# CSS 선택자로 요소 찾기
header = driver.find_element(By.CSS_SELECTOR, "header.main-header")

# XPath로 요소 찾기
submit_btn = driver.find_element(By.XPATH, "//button[@type='submit']")
요소 찾기 팁

1. ID와 같은 고유 식별자를 우선적으로 사용하세요. 가장 빠르고 안정적입니다.

2. 페이지 구조가 변경될 가능성이 있는 경우, XPath보다는 CSS 선택자가 더 안정적입니다.

3. 디버깅에는 Chrome의 개발자 도구를 활용하여 선택자를 테스트해보세요.

요소 찾기 선택자 유형

선택자 설명
By.ID HTML 요소의 id 속성으로 찾기 (가장 빠름)
By.NAME HTML 요소의 name 속성으로 찾기
By.CLASS_NAME HTML 요소의 class 속성으로 찾기
By.TAG_NAME HTML 태그 이름으로 찾기 (예: div, a, button)
By.LINK_TEXT 링크 텍스트가 정확히 일치하는 a 태그 찾기
By.PARTIAL_LINK_TEXT 링크 텍스트의 일부가 일치하는 a 태그 찾기
By.CSS_SELECTOR CSS 선택자로 요소 찾기 (강력하고 유연함)
By.XPATH XPath 표현식으로 요소 찾기 (가장 유연함)
HTML 요소 구조 예시
id="login-form" <form id="login-form" class="user-form">
name="username" <input type="text" name="username" class="input-field">
id="login-btn" <button id="login-btn" type="submit">로그인</button>
</form>

요소를 찾지 못했을 때 처리

# ChatGPT: "셀레니움에서 요소를 찾지 못했을 때 처리하는 코드"
from selenium.common.exceptions import NoSuchElementException

try:
  element = driver.find_element(By.ID, "non-existent-element")
  element.click()
except NoSuchElementException:
  print("요소를 찾을 수 없습니다. 페이지 구조가 변경되었을 수 있습니다.")
  # 대체 동작 수행
  alternative_element = driver.find_element(By.CLASS_NAME, "alt-button")
  alternative_element.click()
